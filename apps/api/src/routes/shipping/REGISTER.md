# REGISTER — shipping / carriers module

Wiring-instructies voor de orchestrator. De feature-agent (Atlas) raakt
`routes/index.ts`, `db/schema/index.ts`, `db/seed.ts`, `lib/env.ts` en de
migrations NIET aan — die worden hier exact beschreven zodat één atomic edit
volstaat.

Alle nieuwe bestanden staan onder:
- `apps/api/src/routes/shipping/**`
- `apps/api/src/db/schema/shipping.ts`
- `apps/api/src/db/seed-shipping.ts`

---

## 1. `apps/api/src/routes/index.ts` — mount op `/shipping`

Import bovenaan (bij de andere route-imports):

```ts
import { shippingRoutes } from './shipping/index.js';
```

Mount (bij de andere `app.route(...)`-mounts, achter dezelfde auth-conventie —
de router zet zelf `requireAuth` op `*`):

```ts
app.route('/api/shipping', shippingRoutes);
```

---

## 2. `apps/api/src/db/schema/index.ts` — re-export schema

Voeg toe bij de andere `export * from`-regels:

```ts
export * from './shipping.js';
```

Dit exporteert `shippingCarriers`, `shipments` + de `ShippingCarrier` /
`NewShippingCarrier` / `Shipment` / `NewShipment` types, zodat ze in de Drizzle
schema-map (gebruikt door `db`/`runInTransactionWithAudit`) zitten.

---

## 3. `apps/api/src/db/seed.ts` — seed-call

Import bovenaan:

```ts
import { seedShipping } from './seed-shipping.js';
```

Aanroep in de hoofd-seed-flow (na `seedChannels()` of waar de andere
optionele seeds staan):

```ts
await seedShipping();
```

Idempotent: checkt per `code` (UNIQUE) en slaat bestaande over. Seedt
sendcloud / myparcel / postnl / dhl als `status='disconnected'`.

---

## 4. `apps/api/src/lib/env.ts` — env vars (allemaal `.optional()`)

GEEN NIEUWE VERPLICHTE ENV VARS NODIG. De module hergebruikt
`CHANNEL_SECRET_KEY` (bestaat al) voor credential-encryptie via `channel-crypto`.

Optioneel — alleen toevoegen als de orchestrator default API-base-URL-overrides
wil ondersteunen (de adapters hebben de officiële URLs hard-coded als default,
dus dit is puur voor toekomstige flexibiliteit en NIET vereist):

```ts
  // Shipping (optioneel — adapters hebben officiële defaults)
  SENDCLOUD_API_BASE: z.string().url().optional(),
  MYPARCEL_API_BASE: z.string().url().optional(),
  POSTNL_API_BASE: z.string().url().optional(),
```

> NB: de huidige adapters lezen deze env vars NIET (base-URLs zijn constanten in
> de adapter-files). Voeg ze alleen toe als je de adapters later wilt laten
> overriden. Niets breekt als je ze weglaat.

## 5. Migration — `apps/api/drizzle/0005_shipping.sql`

Hand-geschreven, PUUR ADDITIEF (zoals 0001–0004; drizzle-kit `db:generate`
faalt op de ESM `.js`-imports). Gebruikt de bestaande `set_updated_at()`-functie
uit 0000. Sla op als `0005_shipping.sql`.

```sql
-- ============================================================
-- Migration 0005 — Shipping / carriers (koppel-klaar)
-- Generated by Atlas (handmatig conform Drizzle-conventies).
--
-- PUUR ADDITIEF — bestaande tabellen/kolommen worden NOOIT aangeraakt.
-- 2 nieuwe tabellen: shipping_carriers + shipments.
-- Hergebruikt set_updated_at() (bestaat al sinds 0000).
-- ============================================================

-- ─── shipping_carriers ───────────────────────────────────────
CREATE TABLE IF NOT EXISTS "shipping_carriers" (
  "id" uuid PRIMARY KEY DEFAULT gen_random_uuid() NOT NULL,
  "code" text NOT NULL,
  "name" text NOT NULL,
  "status" text DEFAULT 'disconnected' NOT NULL,
  "credentials" jsonb,
  "config" jsonb DEFAULT '{}'::jsonb NOT NULL,
  "last_test_at" timestamp with time zone,
  "created_at" timestamp with time zone DEFAULT now() NOT NULL,
  "updated_at" timestamp with time zone DEFAULT now() NOT NULL,
  CONSTRAINT "shipping_carriers_code_unique" UNIQUE ("code")
);

-- ─── shipments ───────────────────────────────────────────────
CREATE TABLE IF NOT EXISTS "shipments" (
  "id" uuid PRIMARY KEY DEFAULT gen_random_uuid() NOT NULL,
  "order_id" uuid NOT NULL REFERENCES "orders"("id") ON DELETE CASCADE,
  "carrier_id" uuid REFERENCES "shipping_carriers"("id") ON DELETE SET NULL,
  "carrier_code" text,
  "tracking_code" text,
  "tracking_url" text,
  "label_url" text,
  "status" text DEFAULT 'pending' NOT NULL,
  "weight_grams" integer,
  "raw" jsonb,
  "created_at" timestamp with time zone DEFAULT now() NOT NULL,
  "updated_at" timestamp with time zone DEFAULT now() NOT NULL
);

CREATE INDEX IF NOT EXISTS "shipments_order_id_idx"
  ON "shipments" ("order_id");

-- ─── updated_at-triggers (bestaande set_updated_at() uit 0000) ──
DROP TRIGGER IF EXISTS shipping_carriers_updated_at ON "shipping_carriers";
CREATE TRIGGER shipping_carriers_updated_at
  BEFORE UPDATE ON "shipping_carriers"
  FOR EACH ROW EXECUTE FUNCTION set_updated_at();

DROP TRIGGER IF EXISTS shipments_updated_at ON "shipments";
CREATE TRIGGER shipments_updated_at
  BEFORE UPDATE ON "shipments"
  FOR EACH ROW EXECUTE FUNCTION set_updated_at();
```

### 5b. Journal-entry voor de migrator

De runner (`drizzle-orm/postgres-js/migrator`) leest `drizzle/meta/_journal.json`.
Voeg een entry toe aan het `entries`-array (na idx 4):

```json
    {
      "idx": 5,
      "version": "7",
      "when": 1749081600000,
      "tag": "0005_shipping",
      "breakpoints": true
    }
```

(`when` mag elke timestamp > de vorige zijn; bovenstaande = 0004 + 1 dag.)

## Samenvatting van endpoints (achter `requireAuth`, gemount op `/api/shipping`)

| Methode | Pad | Doel |
| --- | --- | --- |
| GET    | `/carriers` | list (masked creds + shipment-counts) |
| POST   | `/carriers` | create `{code,name,config?}` (409 bij dubbele code) |
| GET    | `/carriers/:id` | detail |
| PATCH  | `/carriers/:id` | update name/config/status |
| DELETE | `/carriers/:id` | delete (shipments.carrier_id → set null) |
| PUT    | `/carriers/:id/credentials` | encrypt + store + verify → status |
| POST   | `/carriers/:id/test-connection` | verify → status + last_test_at |
| POST   | `/shipments` | create label voor order (guarded → 409 `carrier_not_connected`) |
| GET    | `/shipments?order_id=&status=&limit=&offset=` | list |
| GET    | `/shipments/:id` | detail |
| GET    | `/shipments/:id/tracking` | adapter.getTracking (guarded → 409) |

Error-shapes spiegelen channels: `{error:'invalid_request',details}` / `not_found`
/ 409 `{error:'carrier_not_connected'}`.
